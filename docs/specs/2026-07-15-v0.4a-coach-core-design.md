# v0.4a — Coach core (memory, personalities, thinking modes, ghost mode)

Status: approved 2026-07-15 · Roadmap phase: v0.4 (docs/ROADMAP.md) · Sub-spec A of 3 (A → B → C)

## Goal

The coach remembers who it's coaching, speaks in the tone you pick, thinks as
hard as the question deserves, and can hold throwaway conversations that
don't clutter history.

**Definition of done:** a fact told to the coach in one thread is known in a
new thread; the personality preset changes the coach's tone; a Quick message
answers on the small model and a Deep one on the large model; a ghost thread
disappears within 24 hours without being asked.

## Decisions (with reasons)

- **Memory is a table, not a JSON blob** (`coach_memories`: one row per
  fact, categorized): rows are individually editable/deletable from the
  settings card and by the `forget` tool; a single JSON document would make
  concurrent tool writes and UI edits fight each other.
- **Written by tool + editable in settings** (user choice): the coach calls
  `remember`/`forget` mid-chat; a settings card lists entries with
  edit/delete. Both paths hit the same service module.
- **Injected as a compact block into the system prompt** in
  `coach-context.ts`, capped at 50 entries / ~2000 chars total (oldest
  non-goal entries dropped first past the cap) — the prompt can never bloat
  unboundedly.
- **Two model slots, not a mode enum tied to providers**: `llm_settings`
  gains `modelQuick` + `modelDeep` (migration backfills both from `model`,
  which stays as the legacy fallback) and `defaultMode`. Works identically
  for Anthropic and OpenAI-compatible/Ollama — users type whatever two model
  ids their provider serves. Anthropic defaults suggested in the UI:
  quick `claude-haiku-4-5`, deep `claude-opus-4-8` (per current model
  catalog; Fable-tier excluded — premium pricing and retention requirements
  don't fit a BYO-key self-hosted default).
- **Mode travels with the request**: the chat composer sends
  `mode: "quick" | "deep"` per message (toggle defaults to the user's
  `defaultMode`); `resolveProvider` picks the model. No server-side session
  state.
- **Ghost mode is a thread property** (`ephemeral` boolean), not a separate
  chat surface: purge = `DELETE FROM chat_threads WHERE ephemeral AND
updated_at < now() - 24h` (messages cascade), folded into the existing
  scheduler tick. Ghost threads are excluded from memory writes (`remember`
  becomes a no-op with a note to the model) so throwaway chats can't
  contaminate durable memory.
- **Personalities are preamble presets, not free-text**: `analytical` |
  `encouraging` (default) | `direct`, stored on `llm_settings`
  (`coachPersonality`). Each maps to a short tone paragraph merged into
  `buildSystemPrompt`. The persona's safety rules (never program through
  injury/illness, cite real numbers only) are outside the preset and apply
  to all three.

## Components

### Schema (`src/lib/db/schema.ts` + one generated migration)

- `coach_memories`: id uuid PK, userId FK cascade, `category` enum
  (`goal | injury | race | preference | fact`), `content` text (≤ 280
  chars, enforced in service), createdAt/updatedAt. Index on userId.
- `chat_threads`: add `ephemeral boolean NOT NULL DEFAULT false`.
- `llm_settings`: add `modelQuick text`, `modelDeep text`,
  `defaultMode text enum('quick','deep') NOT NULL DEFAULT 'deep'`,
  `coachPersonality text enum('analytical','encouraging','direct') NOT NULL
DEFAULT 'encouraging'`. Migration backfills modelQuick/modelDeep from
  `model`.

### Memory service (`src/lib/coach-memory.ts`)

- `listMemories(userId)` → rows ordered category, createdAt.
- `saveMemory(userId, category, content)` → insert; enforces 280-char and
  50-entry caps (over cap: reject with a clear error the tool surfaces to
  the model).
- `updateMemory(userId, id, content)` / `deleteMemory(userId, id)` —
  ownership-checked (cross-user: not found).
- `memoryPromptBlock(userId)` → `"## What you know about this athlete"` +
  bulleted entries grouped by category, truncated to ~2000 chars; empty
  string when no rows.

### Tools (`src/lib/tools/remember.ts`, `src/lib/tools/forget.ts`)

- Registered in `registry.ts` like existing tools; scope `write:memory`
  (new scope added to the MCP scope list, granted like `write:wellness`).
- `remember({category, content})` → saveMemory; in ephemeral threads the
  chat route passes `ephemeral: true` in context and the tool returns
  `{saved: false, reason: "ghost thread"}` without writing.
- `forget({id})` → deleteMemory; the memory block in the prompt includes
  ids so the model can reference them.

### Chat pipeline (`src/app/api/chat/route.ts`, `src/lib/coach-persona.ts`, `src/lib/llm-provider.ts`)

- `buildSystemPrompt(opts)` gains `personality` (preamble paragraph per
  preset) and the memory block is appended after the athlete snapshot.
- Request body gains `mode?: "quick" | "deep"` and `ephemeral?: boolean`
  (on thread creation). `resolveProvider(userId, mode)` returns the
  matching model (fallback order: mode slot → legacy `model`).
- Composer UI: Quick/Deep segmented toggle (persists per thread in client
  state, initial value = `defaultMode`); "Ghost" toggle only when starting
  a new thread; ghost threads render with a ghost icon and a "deletes in
  24h" hint, and are listed under a separate collapsed section.

### Settings (`src/components/settings/llm-settings-card.tsx`, new `coach-card.tsx`, `src/app/settings/llm-actions.ts` + new `coach-actions.ts`)

- LLM card: two model fields replace the single one (prefilled with
  Anthropic suggestions when providerType is anthropic), default-mode
  select.
- New Coach card: personality select + memory list (inline edit, delete).
  Server actions validate ownership and lengths.

### Purge (`src/lib/sync/scheduler.ts`)

- `purgeEphemeralThreads()` — one DELETE, called each scheduler tick after
  job processing (same guarded pattern as the morning push hook: failures
  logged, never break the tick).

## Error handling

- Memory caps → tool returns a structured refusal the model can relay
  ("memory full — ask the athlete what to forget"); settings card shows
  count "N/50".
- Unknown mode value or missing mode slot → fall back to legacy `model`;
  never 500 the chat.
- Purge failures are logged and retried next tick (tick cadence already
  exists).

## Testing

- Memory service (pg): caps, ownership isolation (cross-user
  update/delete rejected), prompt-block truncation with hand-built rows.
- Tools (registry test extension): remember/forget round-trip; ghost
  context no-op; scope enforcement via existing MCP security tests
  (`write:memory` denied for read-only token); MCP tool-list count
  assertions updated (9 → 11).
- Provider resolution (unit, mocked settings): quick/deep/legacy fallback
  matrix.
- Persona (unit): each personality preamble present; safety rules present
  in all three.
- Purge (pg): thread older than 24h with ephemeral=true deleted (messages
  cascade), fresh ghost and non-ghost threads survive.
- Smoke: demo stack — settings card saves; chat with mode toggle; ghost
  thread badge.

## Out of scope

Coach-initiated memory pruning; memory embeddings/retrieval (inject-all is
fine at ≤50 entries); per-thread personality; sharing memories across users;
training-plan memory (v0.5).
