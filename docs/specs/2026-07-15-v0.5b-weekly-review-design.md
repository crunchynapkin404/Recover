# v0.5b — Proactive Weekly Review Design

> Sub-project B of v0.5. Depends on v0.5a (Artifacts) for inline chart output.

## Problem

Morning insights give a daily snapshot. Athletes also need a weekly perspective:
"Am I trending up or down? How did this week compare to last?" Currently the
coach only answers if asked. A proactive weekly report closes that loop.

## Architecture

Same pattern as `morning-insight.ts`:

```
Scheduler tick → is it user's review day + hour? → gather 7-day data
  → LLM generates structured review (with render_chart call)
  → write to system thread (kind: "weekly")
  → optionally push notification teaser
```

**At-most-once guard:** `weekly_review_last` timestamp in `dailyMetrics` (or a
new column). Skip if already generated for this ISO week.

## Schema Changes

Add to user preferences (either `notificationPrefs` or `llmSettings`):

```sql
ALTER TABLE notification_prefs
  ADD COLUMN weekly_review_day smallint NOT NULL DEFAULT 1,  -- 0=Sun..6=Sat
  ADD COLUMN weekly_review_hour smallint NOT NULL DEFAULT 7; -- 0-23
```

Add guard column:

```sql
ALTER TABLE daily_metrics
  ADD COLUMN weekly_review_week text; -- ISO week string, e.g. "2026-W29"
```

## Data Gathered

The `generateWeeklyReview(userId)` function gathers:

1. **Load:** sum(load) for each of the last 7 days + same for prior 7 days
2. **Sessions:** count per sport this week vs last
3. **Readiness:** avg readiness this week, trend (improving/declining/stable)
4. **CTL/ATL/TSB:** current values + 7-day change
5. **HRV:** this week's avg vs baseline
6. **Best efforts (if any):** PRs hit this week (from `athlete_curves` cache)
7. **Planned vs actual (v0.5c only):** if calendar integration active, compare
   planned load vs executed — graceful skip until then

## LLM Prompt Structure

```
You are generating a weekly training review for {name}.

## This Week's Data
{structured data block}

## Instructions
- Lead with the headline: was this a bigger/smaller/recovery week?
- Call render_chart with a bar chart: daily load this week vs last week
- Comment on readiness trend and recovery quality
- Flag any PRs or notable sessions
- If CTL is rising, note the adaptation is working
- If HRV is suppressed below baseline, suggest pulling back
- Keep it to 3-4 sentences + the chart. This is a summary, not a lecture.
- End with one actionable suggestion for next week.
```

## Scheduler Integration

In `runSchedulerTick()` (after morning insight logic):

```ts
// Weekly review — check once per tick for each user
const now = new Date();
const dayOfWeek = now.getDay(); // 0=Sun
const hour = now.getHours();

for (const user of activeUsers) {
  if (dayOfWeek === user.weeklyReviewDay && hour === user.weeklyReviewHour) {
    const isoWeek = getISOWeek(now);
    if (user.weeklyReviewWeek !== isoWeek) {
      await generateWeeklyReview(user.id);
      // mark as done for this week
    }
  }
}
```

## Thread Organization

- `chat_threads.kind = "weekly"` — one thread per user, messages accumulate
- Dashboard card shows latest weekly review (same pattern as morning insight card)
- Coach route links: `/coach?thread=weekly`
