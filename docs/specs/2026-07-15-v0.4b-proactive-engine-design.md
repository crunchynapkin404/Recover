# v0.4b — Proactive engine (morning insights + overtraining warnings)

Status: approved 2026-07-15 · Roadmap phase: v0.4 (docs/ROADMAP.md) · Sub-spec B of 3 (A → B → C)

## Goal

The coach speaks first: every morning after the overnight sync there is a
short, data-grounded insight waiting — and when HRV or resting HR trends say
the athlete is digging a hole, the insight says so plainly.

**Definition of done:** a morning insight appears on the dashboard unasked
(all three surfaces: dashboard card, line in the morning push, message in
the coach thread); on a fixture user with 8 suppressed-HRV days the insight
carries an overtraining warning; a day with no LLM key configured still
produces a stats-only card; sync never fails because insight generation
failed.

## Decisions (with reasons)

- **One generation path for both features**: overtraining detection runs
  first as pure functions; its result is handed to the insight writer as
  context. No separate "warning pipeline" — a warning is a
  warning-flavored morning insight (`kind: "warning"`), so there is exactly
  one thing to render, push, and dedupe.
- **Detection is deterministic and unit-testable** (`src/lib/overtraining.ts`),
  using the baselines already stored on `daily_metrics`:
  - HRV suppression: ≥ 7 consecutive days with ln(hrv) < baseline mean − 1
    SD (the band the app already draws).
  - RHR spike: 3-day mean resting HR ≥ baseline mean + 10 bpm.
    Both need ≥ 21 days of history and a non-calibrating baseline; otherwise
    `null` (no warning, no guessing).
- **Insights live in chat** (user choice, all three surfaces): a per-user
  system thread titled "Morning coach" (found-or-created by tag,
  `kind: 'morning'` column on `chat_threads`) gets one assistant message
  per day. The dashboard card reads the latest message from that thread —
  no new insight table; history is chat history.
- **Push rides the existing morning notification** (user choice): the
  readiness push body gains one teaser line (first sentence of the insight,
  clamped). No new push types, no new prefs — if the morning push is off,
  no push (dashboard + thread still work).
- **Generated in the same post-sync hook as the push**, before it (so the
  teaser exists when the push builds): guarded exactly like
  `maybeSendMorningReadinessPush` — errors logged, sync unaffected.
  Deduped one per user per local day (skip if the morning thread already
  has a message today).
- **Degrades without an LLM**: no key / LLM error → the message is a
  deterministic stats summary (readiness, band, TSB, and the warning
  sentence if triggered) built from a template. The card never shows an
  error state. Message metadata
  `{ generated: "llm" | "template", warning: kind | null }` is stored in
  the message's `toolCalls` jsonb — the dashboard card reads `warning`
  from there (amber border), not by re-running detection.
- **Strava exclusion holds**: the insight prompt reuses the existing
  coach-context snapshot builder, which already excludes
  `provider='strava'` rows.

## Components

### Detection (`src/lib/overtraining.ts`)

- `detectOvertraining(days: WellnessDayInput[], baseline: BaselineInput)` →
  `{ kind: "hrv_suppression" | "rhr_spike"; sinceDays: number } | null`.
  Pure; inputs are plain rows (no db).
- Query helper `getOvertrainingStatus(userId)` loads the last 21 days of
  `wellness_daily` + latest `daily_metrics` baselines and calls the pure
  function.

### Insight writer (`src/lib/morning-insight.ts`)

- `generateMorningInsight(userId)`:
  1. Dedupe: morning thread already has a message dated today (local) →
     return `skipped`.
  2. Skip while calibrating or when there is no readiness for today.
  3. Run `getOvertrainingStatus`; build the coach context snapshot.
  4. LLM available → one short completion (≤ ~120 words, persona +
     personality applied, explicit instruction to lead with the warning
     when present). Otherwise template text.
  5. Find-or-create the "Morning coach" thread (`kind='morning'`,
     excluded from the normal thread list's "new chat" flows but openable);
     insert the assistant message; return `{ text, warning }`.
- `getLatestMorningInsight(userId)` → today's message or null (dashboard
  reads this).

### Scheduler hook (`src/lib/sync/scheduler.ts`)

- In the post-sync block, before the push call:
  `await generateMorningInsight(job.userId)` wrapped in its own try/catch
  ("morning insight failed" log line).

### Push teaser (`src/lib/push.ts`)

- `buildMorningPayload` gains optional `insightTeaser?: string`; appended
  as a second body line, clamped to 120 chars.
  `maybeSendMorningReadinessPush` fetches today's insight text (if any)
  and passes the first sentence.

### Dashboard card (`src/components/dashboard/coach-insight.tsx`, wired in `src/app/page.tsx`)

- Renders today's insight (glass card, coach icon; amber border when
  `warning`), links to `/coach?thread=<morning-thread-id>`; dismiss = local
  state only (comes back next day with new content). Absent entirely when
  there is no insight today.
- `/coach` page accepts `?thread=` to open a specific thread (small
  addition to the existing chat interface).

## Error handling

- LLM failure/timeout (10 s cap) → template text, logged warning.
- Detection with insufficient history → null (never a false warning while
  calibrating).
- Hook failures never propagate to the sync job (same contract as the push
  hook).
- Duplicate prevention is by query, not state, so restarts/multiple ticks
  stay idempotent.

## Testing

- `overtraining.test.ts` (unit, fixtures): 7-day suppression triggers at
  exactly 7, not 6; RHR spike at +10, not +9; calibrating/short history →
  null; recovery day resets the streak.
- `morning-insight.test.ts` (pg, mocked LLM): creates thread + message
  once per day (second call skips); warning fixture produces
  warning-flavored text; no-key user gets template message; calibrating
  user gets nothing.
- Push (extend `tests/push-payload.test.ts`): teaser line present and
  clamped; absent when no insight.
- Smoke: demo stack — run the scheduler tick manually, insight card
  renders on dashboard, morning thread visible in coach.

Weekly review (v0.5); push-on-warning outside the morning cycle;
