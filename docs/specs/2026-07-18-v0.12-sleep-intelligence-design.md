# v0.12 — Sleep Intelligence: Design

**Date:** 2026-07-18
**Status:** Approved

## Goal

v0.9.0 deleted the fabricated sleep cards; v0.11 started ingesting real
stage data (Whoop/Oura/Apple Health write `sleep_deep/rem/light/awake_secs`
and `bed_start/bed_end`). This release earns the cards back — but only for
athletes whose provider actually sends the data. Done when: a Whoop/Oura
athlete sees stages and a consistency score; a manual athlete sees exactly
what they saw before (nothing invented); and the dashboard uses a laptop
screen instead of the middle 512px of it.

## Principle

Every number here is gated on real provider data. Stage cards render only
when stage columns are populated; a consistency score needs N nights of
real bed/wake times; chronotype needs weekday _and_ free-day data. Absent
data shows nothing or an honest "needs a sleep-tracking device" note —
never an estimate. This is the same discipline as v0.10's `calibrating`.

## Architecture

### Pure engine — `src/lib/sleep-insights.ts`

No db, no I/O (readiness.ts shape). Inputs are wellness rows already loaded
by the page. Constants exported for tests.

- `stageBreakdown(night)` → `{ deepSecs, remSecs, lightSecs, awakeSecs,
asleepSecs, fractions } | null`. Null unless at least one stage column is
  present. Fractions are of total asleep time; a hypnogram-lite bar renders
  from these.
- `sleepConsistency(nights)` → `{ score, sampleNights } | null`. The
  Sleep Regularity Index idea, simplified to what bed/wake times support:
  the standard deviation of sleep **midpoint** (and of duration) across the
  trailing window, mapped to 0–100 where a perfectly regular schedule
  scores 100 and ≥ `MAX_SD_MINUTES` of midpoint scatter scores 0. Needs
  `MIN_CONSISTENCY_NIGHTS` real bed/wake nights or returns null.
- `chronotype(nights)` → `{ midpointHhMm, socialJetlagMins, weekdayMidpoint,
freeDayMidpoint } | null`. Mean sleep midpoint overall; social jetlag =
  |free-day midpoint − weekday midpoint|. Needs ≥ 2 nights on each side.
- `napAware(nightsForDate)` → sums multiple same-date sleep sessions
  honestly (main + naps), labelling nap contribution — the merge already
  keeps the longest main sleep per date on `wellness_daily`, so "naps" here
  means additional sessions the provider reports separately; v0.12 surfaces
  the summed total and flags when a nap padded it.

Midpoint math works in minutes-from-local-midnight with wrap handling (a
02:00 midpoint is 26·60, not 2·60) so a 23:30→07:30 night has a 03:30
midpoint, not a noon one.

### UI — dashboard sleep section

- **`SleepStagesCard`** (new): a stacked stage bar (deep/rem/light/awake)
  with per-stage minutes and fractions, plus the bed/wake window when
  present. Rendered **only** when `stageBreakdown` is non-null. Replaces
  nothing for manual athletes — it simply doesn't mount.
- **`SleepConsistencyCard`** (new): the 0–100 score with a one-line reading
  ("very regular" / "somewhat irregular" / …) and the sample size. Hidden
  below the minimum nights.
- **`ChronotypeCard`** (new): midpoint-of-sleep and social-jetlag readout.
- The existing `SleepCard` (score, duration, debt, bedtime) stays as the
  always-present summary; the new cards sit beside it when data exists.

### Bedtime target v2 — `sleep-debt.ts`

When the trailing nights carry real `bedStart`, prefer the athlete's actual
median bedtime as the anchor, nudged by outstanding debt, over the pure
wake-time − need arithmetic. The manual `wakeTime` path stays intact for
athletes with no provider bed data (unchanged behavior for them). Additive
signature: an optional `bedtimes` array; absent → today's behavior exactly.

### Desktop shell — `app-shell.tsx` + `SidebarNav`

The phone-stripe-on-a-monitor `max-w-lg` becomes responsive:

- `< lg`: unchanged — `max-w-lg` centered column, floating bottom tab bar.
- `≥ lg`: a persistent left `SidebarNav` (same routes as BottomNav, labelled
  vertically) and a wider content area (`max-w-5xl`) that lets the dashboard
  flow into two columns and charts breathe. BottomNav is hidden at `lg`.
- `noChrome` pages (the dashboard) opt into the wider max-width themselves;
  the shell just stops constraining to 512px and stops reserving bottom-tab
  space at `lg`.

Layout only — no route or data changes. The dashboard's single column
becomes a `lg:grid-cols-2` where it reads well (hero spans full; the sleep,
vitals, and weekly cards tile).

## Testing

- Engine unit tests (`sleep-insights.test.ts`): stage breakdown present/
  absent, midpoint wrap-around, consistency score at regular vs scattered
  schedules and the min-nights gate, social jetlag weekday/free-day split,
  nap summing.
- Bedtime v2: real-bedtimes path vs wake-time fallback, both pure.
- Component tests: SleepStagesCard renders bars from fractions and does not
  mount without stage data; consistency/chronotype hidden states.
- Existing suite stays green; no test depends on external env.

## Out of scope

- Sleep-stage _scoring_ algorithms (we render provider stages, not invent a
  score) — sleepScore stays the provider's number.
- Apnea/SpO2 (no column, no providers wired for it here).
- v0.13 Deep Biology; v0.14 forecasts.
- A full design-system rework of the desktop layout — this is a responsive
  shell, not a redesign; the design-&-UX track carries the rest.
